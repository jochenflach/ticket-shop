ℹ️ Funktionsweise der Stripe-Umschaltung

Im Code ist bereits alles vorbereitet. Solange keine echten Stripe-Umgebungsvariablen hinterlegt sind, schaltet der Shop automatisch in den Zahlungssimulations-Modus. Sobald die Keys eingetragen werden, ist die echte Zahlungsmaske sofort live.

🔑 Schritt 1: Stripe-Konto & API-Keys holen

Stripe wickelt die Kreditkarten-, Lastschrift- und Smartphone-Zahlungen ab und überweist das Geld auf dein Bankkonto.

  1. Erstelle ein Konto auf Stripe.com.
  2. Aktiviere dein Konto für den echten Zahlungsverkehr (oder bleibe oben rechts im Test-Modus für Probezahlungen).
  3. Navigiere im Stripe-Menü auf: Entwickler → API-Schlüssel.
  4. Kopiere dort den Geheimen Schlüssel (Secret Key).

Hinweis zum Key-Format: Im Testbetrieb lautet der Schlüssel z. B. sk_test_51P..., im echten Live-Betrieb beginnt er mit sk_live_....

🌐 Schritt 2: Webhook in Stripe einrichten

Ein Webhook ist eine automatische Benachrichtigung. Sobald ein Kunde bei Stripe bezahlt hat, schickt Stripe ein Signal an unseren Server. Erst dadurch werden die Tickets generiert und per Mail versendet.

  1. Navigiere im Stripe-Dashboard auf Entwickler → Webhooks.
  2. Klicke oben rechts auf Endpunkt hinzufügen.
  3. Trage bei Endpunkt-URL deine Netlify-Scanner-Adresse ein:
    https://wondrous-gumdrop-b59508.netlify.app/api/webhook/stripe
  4. Klicke auf + Ereignisse auswählen, suche in der Liste nach:
    checkout.session.completed (Zahlung erfolgreich abgeschlossen) und hake es an.
  5. Klicke auf Endpunkt hinzufügen.
  6. Klicke unter dem erstellten Webhook auf "Signierschlüssel anzeigen" und kopiere diesen geheimen Schlüssel (beginnt mit whsec_...).

🚀 Schritt 3: Keys in Netlify eintragen

Um die Zugangsdaten sicher zu hinterlegen, tragen wir sie direkt bei Netlify ein (so sind sie für Kunden unsichtbar geschützt).

  1. Logge dich auf Netlify.com ein und wähle dein Projekt "wondrous-gumdrop-b59508" aus.
  2. Gehe im Menü auf Site configuration → Build & deploy → Environment variables.
  3. Klicke auf Add a variable → Create single variable und trage ein:

    1. Stripe Secret Key

    Key: STRIPE_SECRET_KEY Value: sk_test_... (dein Secret Key aus Schritt 1)

    2. Stripe Webhook Secret

    Key: STRIPE_WEBHOOK_SECRET Value: whsec_... (dein Signierschlüssel aus Schritt 2)
  4. Klicke unten auf Save (Speichern).
  5. Gehe im linken Menü auf den Reiter Deploys, klicke rechts auf Trigger deploy → Deploy site. Der Server startet neu und wendet die neuen Live-Zahlungseinstellungen an.

🌐 Schritt 4: Subdomain-Integration (Eigene Webadresse)

Statt den Shop in ein enges iFrame auf deiner Webseite zu quetschen, betreibst du ihn am professionellsten auf einer eigenen Subdomain deiner Haupt-Webseite (z. B. tickets.deine-homepage.de oder shop.deine-homepage.de).

Vorteile der Subdomain-Lösung:

  • Volles Vertrauen: Die Kunden bleiben optisch komplett auf deiner Domain (keine fremde Netlify-Adresse im Browser).
  • Kein iFrame-Frust: Keine lästigen Scrollbalken, keine mobilen Zoom-Probleme oder abgeschnittene Ränder auf Smartphones.
  • Sicherheit & Cookies: Keine Blockaden durch restriktive Drittanbieter-Cookie-Richtlinien (z. B. von Apple Safari), da Subdomains als First-Party eingestuft werden.

Wie es funktioniert:

  1. Subdomain beim Domain-Anbieter anlegen: Logge dich bei deinem Domain-Host (z. B. Strato, 1&1 IONOS, Hostinger) ein und erstelle eine neue Subdomain (z. B. tickets.deine-homepage.de).
  2. CNAME-Eintrag setzen: Konfiguriere für die neue Subdomain einen CNAME-Eintrag und verweise als Zieladresse direkt auf deine Netlify-Adresse:
    wondrous-gumdrop-b59508.netlify.app
  3. Domain in Netlify hinzufügen:
    • Gehe in deinem Netlify-Projekt auf Site configuration → Domain management.
    • Klicke auf Add custom domain und trage deine neue Subdomain (z. B. tickets.deine-homepage.de) ein.
    • Netlify überprüft den CNAME-Eintrag und generiert automatisch das kostenlose SSL-Zertifikat (HTTPS) für deine Subdomain.
  4. Auf der Hauptseite verlinken: Erstelle auf deiner Haupt-Homepage einfach einen auffälligen Button "Tickets kaufen", der direkt zu deiner neuen Subdomain verlinkt.

✅ Checkliste nach der Einrichtung

So überprüfst du, ob alles reibungslos funktioniert:

  • Öffne den Ticketshop und lege ein Ticket in den Warenkorb.
  • Klicke auf den Button "Kostenpflichtig buchen". Du wirst direkt auf die sichere Bezahlseite von stripe.com weitergeleitet.
  • Führe eine Testzahlung durch mit folgenden Testdaten:
    Kartennummer: 4242 4242 4242 4242 Ablaufdatum: Ein beliebiges zukünftiges Datum (z.B. 12 / 29) CVC-Code: Drei beliebige Ziffern (z.B. 123) PLZ: Eine beliebige Postleitzahl (z.B. 12345)
  • Nach der Zahlung wirst du zurück auf die Erfolgsseite des Shops geleitet und siehst dein Ticket.
  • Prüfe im Admin-Dashboard unter Geplante Veranstaltungen → Ticket-Statistiken, ob das Ticket korrekt erfasst wurde und der Umsatz aktualisiert wurde.
